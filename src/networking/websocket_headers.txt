Messages sent through the websocket start with an integer. That specifies the header.

Various kinds of messages can be send through the socket. The header indicates what the message is.

0:
Message contains gamestate. Sent from the server to all clients every tick.

1:
Message contains GameInput. Sent from a client to the server. GameInput can change the state.

2:
Message contains ClientJoin. Sent from the server to a client upon connection.
Contains a single u32, the client's id.
